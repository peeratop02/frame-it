# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

**Frame It** — an iOS (26+) app that wraps a chosen photo in a customizable frame
displaying its capture metadata (device, date, shutter, ISO, focal length, lens, location).
Built with SwiftUI. iPad support and a 3-tier monetization model are planned for later phases.

## Commands

The Xcode project is **generated by XcodeGen** from `project.yml` — never hand-edit
`FrameIt.xcodeproj`. Change `project.yml` (or add/remove files under `FrameIt/`) and regenerate.

```bash
# Regenerate the project after changing project.yml or adding/removing source files
xcodegen generate

# Build (no booted simulator required)
xcodebuild -project FrameIt.xcodeproj -scheme FrameIt \
  -destination 'generic/platform=iOS Simulator' -configuration Debug build

# Run the test suite (needs a concrete simulator destination)
xcodebuild -project FrameIt.xcodeproj -scheme FrameIt \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' test

# Run a single test (Swift Testing): filter by suite/test name
xcodebuild -project FrameIt.xcodeproj -scheme FrameIt \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  test -only-testing:FrameItTests/ExposureFormattingTests
```

If `xcrun simctl list runtimes` shows nothing in a sandboxed shell, the iOS 26 runtime is
still installed — use the `generic/platform=iOS Simulator` destination for builds, and a
named device (`iPhone 17 Pro`) only when running tests or the app.

## Architecture

SwiftUI + **`@Observable` MVVM** (not `ObservableObject`). View models depend on
**protocol-based services** so they can be unit-tested with mocks. Keep files small and
feature-scoped.

- **`FrameIt/App/`** — `@main` entry (`FrameItApp`) and `RootView` (the bottom `TabView`:
  Photos + Settings). The **Editor is not a tab** — it opens as a `fullScreenCover` from a
  photo tap inside the Photos tab.
- **`FrameIt/Features/`** — one folder per screen (`Library`, `Editor`, `Settings`); each has
  a `View` + an `@Observable` view model + small subviews.
- **`FrameIt/Core/`** — domain logic, no SwiftUI:
  - `Photos/` — `PhotoLibraryService` (PhotoKit wrapper, protocol + impl), `PhotoAsset`.
  - `Metadata/` — `MetadataService` (EXIF via ImageIO `CGImageSource` + GPS reverse-geocode
    via `CLGeocoder`), `PhotoMetadata`, `ExposureFormatting` (pure formatters, heavily tested).
  - `Frame/` — `FrameStyle` (the Codable style model), `FrameLayout`, `FrameRenderer`.
  - `Export/` — `ExportService` (save to Photos add-only + share sheet).
- **`FrameIt/Design/`** — `Theme`, `Typography`, `GlassComponents` (reusable Liquid Glass UI).

### Key data flow (editor → export)

`PHAsset` → `PhotoLibraryService` (full-res `Data`) → `MetadataService` (`PhotoMetadata`) →
bound into a `FrameStyle` → **`FramePreview`** SwiftUI view → `FrameRenderer` (`ImageRenderer`,
scaled to the source's pixel size for full-res output) → `ExportService`.

`FramePreview` is the **single source of truth** for how a framed photo looks — it is rendered
both on screen and during export, so the exported image always matches the preview.

## Conventions specific to this project

- **iOS 26+, Liquid Glass only.** There is no material/older-OS fallback path — use
  `.glassEffect()` / `GlassEffectContainer` directly. Wrap multiple glass siblings in a
  `GlassEffectContainer`; reserve glass for toolbars, the editor control dock, cards, and
  interactive controls (not every view).
- **Swift 5 language mode** is set in `project.yml` to keep builds green around PhotoKit/ImageIO
  (async/await, `@Observable`, `Sendable` are all still available). Upgrading to Swift 6 is a
  deliberate later task, not an accident to "fix".
- **No location permission is requested.** GPS comes from the photo's own embedded EXIF; only
  network reverse-geocoding (`CLGeocoder`) is used to turn coordinates into a place name.
- Honor the installed HIG skill (`.agents/skills/ios-design-guidelines`): 44pt touch targets,
  safe areas, primary actions in the thumb zone, semantic colors, Dynamic Type, VoiceOver.
- `SwiftUI.Tab` is a builder type — **do not name local enums `Tab`** (shadows the view).
  `RootView` uses `Section` for this reason.

## Deferred phases (architected for, not yet built)

Templates (SwiftData `@Model` wrapping a `FrameStyle`, bulk apply, Templates tab) → iCloud sync
(CloudKit private DB) → StoreKit 2 monetization (Free / one-time / subscription tiers) → iPad
(`NavigationSplitView`). `FrameStyle` is `Codable` from day one so templates drop in cleanly.
